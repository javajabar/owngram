# Решение проблемы с доступом без VPN

## Проблема
Сайт не загружается без VPN, вероятно из-за блокировки Supabase или других сервисов.

## Решения

### 1. Использовать альтернативные STUN серверы
✅ **Уже добавлено в код** - теперь используются несколько STUN серверов, включая альтернативные, которые работают без VPN.

### 2. Настройка Supabase для работы без VPN

#### Вариант A: Использовать кастомный домен
1. В Supabase Dashboard → Settings → API
2. Настройте кастомный домен (Custom Domain)
3. Это поможет обойти блокировки

#### Вариант B: Использовать прокси/CDN
1. Настройте Cloudflare или другой CDN перед Supabase
2. Это может помочь обойти региональные блокировки

### 3. Проверить блокировку в вашем регионе

#### Проверьте доступность:
```bash
# Проверьте доступность Supabase
curl -I https://your-project.supabase.co

# Проверьте STUN серверы
ping stun.l.google.com
```

### 4. Настройки браузера/системы

#### Если блокируется на уровне DNS:
- Используйте публичные DNS (8.8.8.8, 1.1.1.1)
- Или настройте DNS-over-HTTPS

#### Если блокируется на уровне провайдера:
- Используйте VPN (как сейчас)
- Или настройте прокси на уровне приложения

### 5. Альтернативные решения

#### Использовать другой хостинг для Supabase:
- Supabase имеет несколько регионов
- Попробуйте создать проект в другом регионе (если доступно)

#### Использовать self-hosted Supabase:
- Разверните Supabase на своем сервере
- Это даст полный контроль над доступностью

## Рекомендации:

1. **Краткосрочное решение**: Продолжайте использовать VPN
2. **Долгосрочное решение**: Настройте кастомный домен для Supabase или используйте прокси/CDN

## Проверка работы:

Откройте консоль браузера (F12) и проверьте:
- Нет ли ошибок подключения к Supabase
- Работают ли STUN серверы (в Network tab)
- Есть ли ошибки WebRTC соединения
