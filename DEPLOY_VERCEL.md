# Инструкция по деплою на Vercel (БЕСПЛАТНО)

## Шаги для деплоя:

### 1. Создайте аккаунт на Vercel
- Перейдите на https://vercel.com
- Нажмите "Sign Up"
- Войдите через GitHub (рекомендуется)

### 2. Подключите репозиторий
- После входа нажмите "Add New Project"
- Выберите ваш репозиторий `OwnGram` (или `owngram`)
- Vercel автоматически определит, что это Next.js проект

### 3. Настройте Environment Variables
В разделе "Environment Variables" добавьте:
- `NEXT_PUBLIC_SUPABASE_URL` - ваш Supabase URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - ваш Supabase anon key

### 4. Настройки проекта
- **Framework Preset**: Next.js (автоматически)
- **Root Directory**: `./` (по умолчанию)
- **Build Command**: `npm run build` (по умолчанию)
- **Output Directory**: `.next` (по умолчанию)
- **Install Command**: `npm install` (по умолчанию)

### 5. Деплой
- Нажмите "Deploy"
- Vercel автоматически соберет и задеплоит проект
- После деплоя вы получите URL вида: `your-project.vercel.app`

## Преимущества Vercel:
✅ **Полностью бесплатно** для личных проектов
✅ **Отличная поддержка Next.js** (создан командой Next.js)
✅ **Автоматические деплои** при push в GitHub
✅ **Быстрая работа** (Edge Network)
✅ **HTTPS из коробки**
✅ **Неограниченное количество проектов** на бесплатном плане

## Лимиты бесплатного плана:
- 100 GB bandwidth в месяц
- 100 deployments в день
- Неограниченное количество проектов
- Автоматические SSL сертификаты

## После деплоя:
1. Ваш сайт будет доступен по адресу `your-project.vercel.app`
2. Каждый push в `main` ветку автоматически создаст новый деплой
3. Вы можете настроить кастомный домен в настройках проекта

## Если возникнут проблемы:
- Проверьте, что все environment variables добавлены
- Убедитесь, что репозиторий публичный или вы дали доступ Vercel
- Проверьте логи сборки в Vercel Dashboard
